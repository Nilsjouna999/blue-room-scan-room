## (a) THE CLASS / KIND LINE — unified vocabulary

### The reconciliation problem

Two vocabularies collided in the research:

- **Subtraction Gate spine (repo-side, wins on law):** kind-words must be **frame-geometry nouns** — `Standoff · Edge · Figure-Ground · Light-Fit · Vector-into-Void`. These name a **relation the frame stages**, so they are structurally person-safe (RULE 0 / BR-AURA-RESEARCH.md:5: "a relation the FRAME stages, not a charge a person emits").
- **GPT docs 01/04 (research-side):** the *charged* contradiction-pair — `Warm Refusal · Quiet Labour · Cold Exit` — modifier + noun, warmth-word bent by a cold-word. This is what gives the line its verdict-heat. But its nouns (refusal, labour, mercy, custody) are **social/institutional**, and — per O1 (d) + BR-S109 — a bare 2-word tag row rendered verbatim is the **KILLED "Fit + Aura code/tag-row" form** and reads as a *second Source Record* / filing address.

**These are not rivals — they are two LAYERS of one label.** The frame-geometry noun is the **architecture** (what kind of relation, schema-legal, machine-constrainable). The contradiction pair is the **register** (how it's spoken, authored, charged). H4 §VII already reached this: refusal/labour are "class-LINE labels, not architectural relations." O4 formalizes the split.

### The unified two-tier kind vocabulary

**Tier A — the KIND (schema-constrained, machine-selectable, one of exactly five).** This is the enum stored in data and the semantic anchor the band rides. It is **never rendered as a bare chip** (that's the killed tag-row). It governs which visual residue mode and which contradiction-pair family are legal.

| KIND (enum) | The relation it names | Frame-geometry test (why it's safe) | Doc-02 visual residue |
|---|---|---|---|
| **Standoff** | Two co-present masses hold, neither yielding | opposition/suspension between elements | off-center tension line |
| **Edge** | Warmth/interior meets a cold or sealed boundary | a seam/threshold/crop divides two zones | one-seam burn |
| **Figure-Ground** | Subject dissolves into / is dwarfed by its field | subject-to-container ratio | ghost smear (thin) / type-burn |
| **Light-Fit** | An illuminant organizes a space not built for it | light source vs. surface mismatch | one-seam burn / ghost smear |
| **Vector-into-Void** | A directed motion fires past the frame edge | eye-line/gesture axis exits the frame | ghost smear (directional) |

**Tier B — the SPOKEN class line (authored contradiction pair, the register layer).** One modifier bending one noun, warmth→cold, both terms drawn from the **frame/optics/procedure** register (never social-moral, never emotion). This is the line a human writes and the reader sees. It must be a **legal instance** of its Tier-A kind.

**The house form for Tier B:** `[warm/legible modifier] + [cold/frame noun]`, 2 words default, noun-ending fragment, hardest word last (S1 §1). The modifier is the "pretend" (S1 §3), the noun is the reveal.

### The rule that keeps Tier B safe (this is the load-bearing fix)

The GPT bank leaks two ways — O4 closes both **at the schema**:

1. **Noun register lock.** Tier-B nouns come from **frame-geometry / optics / procedure**, NOT from doc 01's moral-social bank. Banned as Tier-B nouns: `refusal, mercy, labour, custody, devotion, debt, pardon, witness, tax, severance` used as *bare* social claims. (They can survive only when the modifier pins them to the frame — see below — but the safer default is to reach for a frame noun.)
2. **Emotion-terminal ban.** The "hardest word last" rule (S1 §2) is constrained: the terminal noun is a **frame/procedure texture**, never a **named feeling**. `recoil, shame, loneliness, longing, dread` are dead as Tier-B nouns — this catches S1's own flagged live leak ("recoil," doc 01 line 31). Screen every Tier-B closing word against BR-S113's kill-list before it ships.

**Worked reconciliation — "Warm Refusal" (GPT's headline example) survives, barely, and only reframed:**
- As GPT wrote it, "refusal" is a *social act* → drifts toward the person's intent. **Under O4 it is legal ONLY because it names the GESTURE-EVENT allowed by RULE 0's one allowance** ("performed gesture-intent — the palm stops the lens"). The *frame* refuses (the raised palm halts first contact); the *person* is not characterized. Provenance = `src.stance` "raised flat to the lens — half stop-sign."
- Its Tier-A KIND is **Edge** (warm cabin meets cold window seam) — NOT "Refusal" (which is not a frame-geometry relation).
- So: **KIND = Edge; class line = "Warm Refusal."** The enum is safe-by-construction; the spoken pair carries the heat. That is the marriage.

**Preferred Tier-B noun bank (frame/optics/procedure, screened through COPY_SYSTEM §3 preferred lexicon):**
`stop · hold · edge · seam · exit · void · field · band · rim · brace · cut · reach · channel · cradle · surround · glare · glass · frost · burn · drag · trace`

## (b) THE VERDICT LINE — the spec

The one authored sentence sitting beside the band+kind. This is where doc 01 and doc 05 do their real work.

**Hard spec (S1 §1/§5, ratified):**
1. **2 beats.** (3 only if beat three is an escalating hammer-blow; never 4.)
2. **6–14 words** (15–16 ceiling only if the extra length does *reversal* work, not explanation).
3. **One contradiction**, not two stacked.
4. **Warmth → cold**: beat one is the socially legible surface, beat two the colder frame-fact.
5. **Hardest word last** — and per (a) it is a **frame/procedure noun, never a feeling**.
6. **Hinge hidden in verbs where possible** (doc 05 Pattern B / doc 02 residue verbs): prefer `holds / keeps / leaves / lets / cools / drags / cuts / thins` over an announced "reads as." Explicit "reads as" is the reliable default when the line needs unambiguous critical framing; the verb-hidden form is the ceiling register.
7. **No named-emotion payload, anywhere** — not in beat one, not behind a hinge. This is the BR-S113 / kill-list wall and it is absolute.

**The authoring question (S1 §3 — hand this to the human author verbatim):**
> *"What is this frame pretending to be, before it reveals what it actually is?"*
> Beat one = the pretend (the legible surface). Beat two = the reveal (the colder frame-fact).

**The mandatory safety pass (run AFTER craft, every line — O1 (e) three gates):**
- **Re-authorable cue?** The line keys on a performed cue (gesture, gaze, crop, light, pose), not body/face/build.
- **Grammatical subject?** The subject is the *photograph* (frame/crop/gesture/light/setting), never the person. Prefer nouns `gesture · hand · palm · crop · frame · edge · glass · cabin · light · field · shell · path` over the weak noun "subject" (S5 §4: "subject" still pulls toward biography).
- **Connotation-strip?** Remove the photo-noun; if the remaining predicate describes a person's *stable disposition*, kill it.
- **Swap test (S5 §6):** would the line still work with a different sitter in the same pose/crop/light? If no, it's diagnosis.
- **Two-second test (S5 §6):** could a viewer plausibly register the named effect within 2 seconds, with no biography/trust/empathy? If no, it's over the line.

## (c) THE BRIDGE-PHRASE HIERARCHY (as adopted)

**Backbone = the pivot, not the phrase.** The grammatical move (`is → reads-as`) is what makes a read safe, per S5 §1 and BR-AURA-RESEARCH §2. Adopted ranking (S5 §2, repo-screened):

| Rank | Bridge | Adopted use in Aura | Note |
|---|---|---|---|
| 1 | **verb-hidden hinge** (`holds / keeps / leaves / cools / drags / cuts`) | **Ceiling register — the default to reach for.** Hides the hinge inside the verb; reads most expensive; matches doc-02 residue verbs. | "The window keeps one edge cold." |
| 2 | **reads as** | **Reliable default** when the line needs explicit critical framing. | Don't repeat every card — formula fatigue (S5 risk #1). |
| 3 | **plays as** | performance / pose / withholding reads only | mannered if overused |
| 4 | **lands as** | a verdict-final beat *after* cues are established | weak if it opens the line |
| 5 | **carries** | sustained tonal beat, must be pinned to a concrete cue | soft without a cue |
| — | **feels like** | **avoid** — vibes-adjacent, cliché-prone | S5 rank 7 |

## (f) THE FREE-vs-DEVELOP COPY SPLIT

Per ground-truth #8 (develop = a *mode* not a *grade*; free is complete). The register differs by what is *named*, not by quality:

| | **FREE** (partial record / undeveloped layer) | **DEVELOP** (full artifact minted) |
|---|---|---|
| **What it shows** | the **parts / gestalt** — the co-present elements and a first-glance whole | the **relation named + ranked** — the band + kind + verdict line |
| **Copy shape** | one **gestalt line**: names the two elements present, does NOT resolve their relation | the full capstone: ordinal **band** + **KIND** + authored **verdict line** |
| **Grammar** | inventory / naming register (COPY_SYSTEM preferred lexicon: *signal, frame, gesture, field*) | verdict register (this doc's §b spec) |
| **SRC-01 example** | "A raised palm and a fjord window, both in one cabin frame." *(parts, unresolved)* | Charged · Edge · Warm Refusal — "The palm holds you at the glass; the cabin keeps the warmth behind it." |
| **Free-state words** | good: *partial record · undeveloped layer · archive pull · development pending*; bad: *locked · unlock · premium* (COPY_SYSTEM §3) | — |

**Rule:** the free line **must not** pre-spoil the relation — it lists the tension's two poles ("a palm AND a window") but leaves the *edge between them* unnamed. Develop names the edge, ranks it on the band, and writes the verdict. That is the develop *reveal*, and it is why free stays honest-complete rather than crippled.
