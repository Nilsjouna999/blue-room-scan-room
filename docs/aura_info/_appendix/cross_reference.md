## CONFLICTS (for O6/F1 — do not resolve here)

1. **Doc 03's proposed replacement law vs. BR-S113.** Doc 03 argues the blanket ban is "too broad" and proposes naming the depicted person's gaze/pose/effect directly. Ground truth #2 already rules on this (REJECTED), but the conflict must stay visible in the final doc as "considered and rejected," not silently omitted, since doc 03 also contains material (rewrite mechanics, three-tier boundary, swap test) that DOES survive — a careless synthesis could accidentally re-import the killed law along with the surviving mechanics.

2. **Doc 03's own worked "safe" rewrites contain named-emotion payloads the kill-list bans.** E.g., "make the image land as loneliness," "the hush and recoil of shame," "flatten the room's energy... reads as bored," "sink the image into heaviness." These sit inside a table doc 03 presents as its safety solution, but they violate the kill-on-sight rule (named-emotion-behind-a-hinge) that ground truth #2 explicitly preserves from `BR-AURA-RESEARCH.md`. Net effect: roughly half of doc 03's own example table is not usable verbatim. Needs O6/F1 sign-off on which specific rows (if any) survive a second pass, or whether the whole table should be marked "mechanics only, examples non-authoritative."

3. **Canvas color logic: black-field-of-disappearance (doc 02) vs. warm-silver archive palette (doc 04).** Not a locked-law conflict, but a live open design question with no resolution in either doc, and no repo anchor yet confirms which wins. Likely both are right at different scopes (mark vs. plate chrome) but this needs an explicit call, not an assumption, before CSS is written.

4. **Doc 04's 11-field schema vs. the Subtraction Gate's 3-field thin-plate spine.** Doc 04 predates the Gate verdict entirely and was scoped for a much bigger plate. The Gate's own risk note ("decay-to-Frame-Event if aura[] chips render verbatim") appears to directly warn against doc 04's own shipped chip-row plan. This is the highest-stakes unresolved conflict in the matrix — it determines how much of doc 04's implementation plan survives contact with the Gate verdict. Recommend O6/F1 treat doc 04 as "architecture donor, not schema authority" pending an explicit ruling.

5. **No research-doc coverage of `src.stance` provenance or null-relation authoring** (Gate conditions a and b). Not a disagreement so much as an evidence gap — flagging so another agent doesn't assume doc 04's schema silently satisfies conditions it was never designed against.
