# Blue Room: A Three-Plane Depth & Accent-Economy System for Dark "Archive Luxury"

## TL;DR
- **Stop lightening surfaces to show elevation, and stop spreading accent everywhere.** Build depth from three *materially distinct* treatments — a vignetted ROOM (radial light pool, no borders), flat PLATES (one background step + a hairline white border at ~6–8% opacity, near-zero shadow), and a raised ARTIFACT (multi-layer black shadow + inset top highlight + restrained color glow) — all lit from one top light source (~12 o'clock). This is the single change that fixes the "everything on one z-plane" problem.
- **Run accent like currency, not paint:** cap each viewport at roughly 3 accent instances (1 primary action + 1 hero number + 1 keyword), keep accent off all borders/bars/badges by default, and let only ONE per-source palette "speak" at a time (copper-warm OR frost-cool), neutralizing the other to grey and reserving violet/periwinkle as the shared bridge tone.
- **The card reads as a physical object** through the pokemon-cards-css technique stack (pointer-tracked gradients set to `color-dodge`/`overlay`, masked sparkle/foil layers, a moving specular glare, plus frame bevels that imply thickness) — and Warm Glass Copper vs Cold Prism Frost become believable *materials* by differing in hue, blend mode, glare softness, and edge treatment rather than just swapped colors.

## Key Findings

**1. The canonical dark-elevation model (Material) is the wrong default for luxury — invert it selectively.** Google's Material dark theme communicates elevation by compositing a semi-transparent white overlay over a `#121212` base, increasing opacity with height: the documented scale is 0dp = 0%, 1dp = 5%, 2dp = 7%, 3dp = 8%, 4dp = 9%, 6dp = 11%, 8dp = 12%, 12dp = 14%, 16dp = 15%, 24dp = 16% (generated by Material's `ElevationOverlayProvider.calculateOverlayAlpha` equation, which produces higher `colorOnSurface` alpha at higher elevation). The rationale Google gives is explicit: *"Shadows are less effective in an app using a dark theme, because they will have less contrast with the dark background colors and will appear to be less visible. In order to compensate for this, Material surfaces in a dark theme become lighter at higher elevations, when they are closer to the implied light source. This is accomplished via elevation overlays, which are semi-transparent white (colorOnSurface) overlays."* Apple's HIG uses the same logic with two named tiers — "base" (dimmer, recedes) and "elevated" (brighter, advances) — switching automatically for popovers/modals. **For Blue Room, "lighter = higher" partially breaks the mood:** in a museum/scan-room metaphor the dramatic object is the one catching a *spotlight* in an otherwise dark room, not the one painted a lighter grey. So we keep a *small* surface-lightening step for plates (legibility + hierarchy) but make the artifact pop through light/shadow/glow, not through being the brightest grey on screen. This is the same conclusion premium dark systems reach.

**2. Shadows on near-black must be layered, tinted, and paired with a lit edge.** A single `0 3px 3px rgba(0,0,0,0.2)` is nearly invisible on `#0c0d10` and "looks like a blurry border." The fix is Tobias Ahlin's layered technique (echoed by Josh Comeau): stack 3–5 shadows at doubling offset/blur — verbatim, `.shadow-5 { box-shadow: 0 1px 1px rgba(0,0,0,0.12), 0 2px 2px rgba(0,0,0,0.12), 0 4px 4px rgba(0,0,0,0.12), 0 8px 8px rgba(0,0,0,0.12), 0 16px 16px rgba(0,0,0,0.12); }`, each layer doubling the previous. On dark backgrounds two more rules apply: (a) make the shadow color *darker than the background* (push toward pure black, never grey), and (b) pair it with a **highlight + border + shadow triple** — a 1px inset top highlight in white at low opacity simulates the top edge catching light, a hairline border defines the edge, and the soft outer shadow grounds it. Radix Themes ships exactly this pattern (see Details).

**3. There is a real, shippable token set from the best dark systems.** Radix Themes' dark shadow tokens are published in their source and are the strongest reference for `#0c0d10`-class backgrounds. They combine a hairline ring (`0 0 0 1px gray-a6`), graduated black layers, and — for the inset/recessed token — top-edge inset highlights. Linear's system (near-black `#08090a` base, `#0f1011` panels, accent indigo `#5e6ad2`, borders as transparent white at ~2–8%) is the canonical "luxury near-black." Vercel/Geist proves the restraint extreme: borders are white at ~8%, "no color for decoration," accent appears only when it carries meaning. Spotify proves the accent-economy point on `#121212`: a single green reserved for the play button and active states — note Spotify uses #1DB954 as its core brand green and #1ED760 as the lighter logo/UI tint, both against the #121212 base and #ffffff per developer.spotify.com.

**4. Light logic: one top source, inner highlight = catches light, outer glow = emits light.** UI convention assumes light from the top (~90–120°): top edges get a lighter inner border/inset highlight, bottom edges shadow. Semantically, an **inner top highlight** = the object passively catches ambient room light (use on plates and the card frame); an **outer glow** = the object actively emits light (reserve for the artifact and "rare/paid" states). Award-style dark hero staging uses radial-gradient "spotlight pools" behind the hero object (e.g. `radial-gradient(circle, transparent 160px, rgba(0,0,0,0.85) 200px)` for a spotlight; inverted for a glow pool) — exactly the museum/gallery lighting metaphor Blue Room wants.

**5. Digital cards read as physical via the pokemon-cards-css stack.** Simon Goellner's (simeydotme) open project recreates Pokémon holo foils purely in CSS using: pointer-tracked CSS custom properties driving 3D tilt; multiple background gradients whose `background-position` shifts with cursor x/y; blend modes (`color-dodge` for holo shine, plus `overlay`, `color-burn`/`hard-light` for glitter); a masked sparkle/foil texture layer (an SVG-noise or glitter `.webp`); and a radial "glare" hot-spot that follows the pointer and is clipped to the card window. NBA Top Shot uses a comparable idea in 3D — each Moment is, per its official site, "designed in an interactive 3D cube with 8 corners that you couldn't bend if you tried," with lighting and parallax. Material thickness comes from frame treatments: bevels, dual borders, and an inner shadow at the frame edge.

**6. Accent economy: 70–80/10–20/<10, and accent means "act/important," never "decorate."** The 60-30-10 rule, adapted to dark UIs, flips toward ~70% dark surface / 20% mid grey / <10% accent. The governing principle across editorial/premium sources is *contrast through scarcity*: "If your accent color appears on icon backgrounds, card borders, or section headings, it loses its signal. Accent means 'take action here.' Keep it exclusive." Practitioner guidance for two-action screens is to split the 10% (e.g., 5% primary / 5% secondary) but never let two accents fight at equal weight. For Blue Room's two per-source palettes, the workable rule is **one palette speaks per context**, the other is neutralized to grey, and violet/periwinkle is the shared bridge.

**7. Glow reads premium when it's large-radius, low-opacity, desaturated, and slow.** Cheap neon = tight, saturated, high-opacity (`0 0 10px #0ff, 0 0 20px #0ff`). Premium = a large, soft color pool at low opacity (color-mix ~8–20%), often a breathing animation on a 4–7s ease cycle, animated via opacity on a pseudo-element — never animating `box-shadow` directly. As Tobias Ahlin notes, "animating a change of box-shadow will hurt performance"; the fix is to "animate the opacity of a pseudo element," which holds "a solid 60 FPS." The "rare artifact" feeling should come from *more developed depth and light* — a deeper shadow, a wider glow pool, a brighter lit edge — not particle clutter.

## Details

### A. The Three-Plane Depth System (drop-in tokens, calibrated for `#0c0d10`)

**Global light rule:** one light source at top-center (12 o'clock). Every inset highlight sits on the TOP edge; every drop shadow falls DOWNWARD (positive Y), with no horizontal offset. Never contradict this anywhere.

```css
:root {
  /* ROOM — the deepest plane (page) */
  --room-bg: #0a0b0d;                 /* near-black, between #08090b and #14161b */
  --room-floor: #08090b;              /* darkest, for vignette edges */
  --room-rise: #14161b;               /* lightest room tone, behind hero */

  /* PLATE — document/dossier panels */
  --plate-bg: #101216;                /* ~1 small step above room (legibility) */
  --plate-bg-2: #14161b;              /* nested/secondary plate */
  --plate-border: rgba(255,255,255,0.07);     /* hairline, 6–8% white */
  --plate-border-strong: rgba(255,255,255,0.10);
  --plate-top-highlight: rgba(255,255,255,0.045); /* catches room light */

  /* ARTIFACT — the card & key stat moments */
  --artifact-bg: #15171c;
  --artifact-edge-light: rgba(255,255,255,0.10); /* lit top edge */
  --artifact-border: rgba(255,255,255,0.08);
}
```

**ROOM (deepest — vignette/radial pool, NO border, NO shadow).** The room is defined purely by light, like a dark stage. Put a faint radial "rise" of light behind the hero (the card), and a vignette that darkens the page edges:

```css
.room {
  background:
    radial-gradient(120% 80% at 50% 38%,
      var(--room-rise) 0%, var(--room-bg) 45%, var(--room-floor) 100%);
}
.room::after { /* vignette */
  content:""; position:fixed; inset:0; pointer-events:none;
  background: radial-gradient(130% 100% at 50% 50%,
    transparent 55%, rgba(0,0,0,0.55) 100%);
}
```

**PLATE (flat document — background step + hairline border + optional top highlight, minimal shadow).** Plates are paper on a table: they sit, they don't float. No drop shadow by default (or an almost-imperceptible one). The defining cues are the small background step and the hairline border, optionally with a 1px inset top highlight to imply the room light grazing the top edge.

```css
.plate {
  background: var(--plate-bg);
  border: 1px solid var(--plate-border);
  border-radius: 10px;
  /* lit top edge only — no outer shadow for the "document on table" read */
  box-shadow: inset 0 1px 0 0 var(--plate-top-highlight);
}
/* a plate that should feel slightly raised (a module, not a document) */
.plate--raised {
  box-shadow:
    inset 0 1px 0 0 var(--plate-top-highlight),
    0 1px 2px rgba(0,0,0,0.4),
    0 4px 12px -4px rgba(0,0,0,0.5);
}
```

**ARTIFACT (object — strong multi-layer shadow + lit top edge + material/glow).** The card and key stat moments are the only things that truly float and emit. Full triple: inset top highlight (lit edge) + hairline border + a deep, layered, near-black shadow, plus the per-source glow pool (see §C).

```css
.artifact {
  background: var(--artifact-bg);
  border: 1px solid var(--artifact-border);
  border-radius: 16px;
  box-shadow:
    inset 0 1px 0 0 var(--artifact-edge-light),   /* top edge catches light */
    0 2px 4px rgba(0,0,0,0.5),                     /* contact shadow */
    0 8px 16px -2px rgba(0,0,0,0.55),
    0 20px 40px -8px rgba(0,0,0,0.6),              /* ambient lift */
    0 40px 80px -16px rgba(0,0,0,0.65);            /* deep room shadow */
}
```

The instant-read hierarchy: room has **no edge** (pure light), plate has a **hard hairline edge but stays flat**, artifact has a **lit edge + deep shadow + glow**. Three different physical behaviors, not three shades of grey.

### B. Real multi-layer shadow & border recipes (named references)

**Radix Themes dark tokens (verbatim from their published `shadow.css`)** — the best calibrated reference for near-black surfaces. Note the structure: every level leads with a `0 0 0 1px` hairline ring (the border), then graduated black layers; the inset token (`--shadow-1`) builds the recessed/lit-edge look:

```css
/* Radix Themes — dark mode (from radix-ui/themes shadow.css) */
--shadow-1:  /* inset / recessed */
  inset 0 -1px 1px 0 var(--gray-a3),
  inset 0 0 0 1px var(--gray-a3),
  inset 0 3px 4px 0 var(--black-a5),
  inset 0 0 0 1px var(--gray-a4);
--shadow-2:  /* resting card */
  0 0 0 1px var(--gray-a6),
  0 0 0 0.5px var(--black-a3),
  0 1px 1px 0 var(--black-a6),
  0 2px 1px -1px var(--black-a6),
  0 1px 3px 0 var(--black-a5);
--shadow-3:
  0 0 0 1px var(--gray-a6),
  0 2px 3px -2px var(--black-a3),
  0 3px 8px -2px var(--black-a6),
  0 4px 12px -4px var(--black-a7);
--shadow-4:  /* raised panel */
  0 0 0 1px var(--gray-a6),
  0 8px 40px var(--black-a3),
  0 12px 32px -16px var(--black-a5);
--shadow-5:
  0 0 0 1px var(--gray-a6),
  0 12px 60px var(--black-a5),
  0 12px 32px -16px var(--black-a7);
--shadow-6:  /* modal / highest */
  0 0 0 1px var(--gray-a6),
  0 12px 60px var(--black-a4),
  0 16px 64px var(--black-a6),
  0 16px 36px -20px var(--black-a11);
```

The lesson to port: lead with the 1px hairline ring, then layer increasingly-blurred black with **negative spread** on the larger layers to keep the shadow tight under the object rather than haloing sideways.

**Linear (the canonical luxury near-black).** Base background `#08090a`, panel surface `#0f1011`, primary text `#f7f8f8`, muted grey `#8a8f98`, accent indigo `#5e6ad2` (hover `#828fff`). Borders are **transparent white at ~2% / 5% / 8%** opacity tiers so they adapt to any surface. (Linear does not publish production CSS; the surface/border/shadow specifics circulate via community replica design files and color-extraction tools like Mobbin, while `#08090a` and `#5e6ad2` are independently corroborated — treat the deepest values as confirmed and the elevated-surface hexes as close replicas.) Linear's documented elevation philosophy is "luminance stacking": default cards carry **no drop shadow** at all; only floating elements (menus, command palette, modals) get shadow, typically soft black layers plus an `inset 0 0 0 1px rgba(255,255,255,~0.08)` lit hairline — the exact triple recommended above.

**Vercel/Geist.** Borders white at ~8%; surface lift comes from `#ffffff`-ish surface on `#fafafa`-ish canvas (light) / near-black tiers (dark) rather than shadows; "no color for decoration." Filled primary action is near-black `#171717`, never the blue accent `#0070f3` — a strong model for keeping accent off CTA *fills* if you want maximum restraint.

**shadcn/ui dark defaults** (HSL, no wrapper): `--background: 240 10% 3.9%` (blue-tinted near-black), `--card`/`--popover` a 3–5% lightness bump above background, `--border` in the `240 3.7% 15.9%` range. Their guidance is explicit and matches Blue Room's plan: don't use pure black; cards should be only "slightly lighter than `--background`… a 3–5% lightness bump is enough. More than that and your cards look like they're floating too high."

**Border-vs-shadow decision table for Blue Room:**
- **Flat document (most dossier plates):** background step + 1px border @ `rgba(255,255,255,0.06–0.08)` + optional inset top highlight. No outer shadow.
- **Raised module (interactive cards, the analysis/reading panels if they should lift):** above + a 2-layer soft shadow (`0 1px 2px / 0 4px 12px -4px` black @ 0.4–0.5).
- **Object (the trading card, key stat moment):** full 4–5-layer near-black shadow + lit top edge + glow.
- **Emissive/active (paid/rare, focused CTA):** add the outer glow pool; never use glow on a resting document.

### C. The Card-Material Playbook: Warm Glass Copper vs Cold Prism Frost

Use the pokemon-cards-css architecture (simeydotme): a pointer-driven `mousemove` handler writes CSS custom properties (`--pointer-x`, `--pointer-y`, `--bg-x`, `--bg-y`, `--glare-opacity`), which drive (1) a 3D `rotateX/rotateY` tilt on a wrapper, (2) the `background-position` of stacked gradient layers, and (3) a radial glare hot-spot. Layer order, bottom→top: card art → **foil/holo gradient layer** (blend `color-dodge`) → **sparkle/texture mask** (blend `overlay`/`color-burn`, masked to the foil area) → **glare** (radial highlight following pointer, clipped to the card window) → frame/bevel. Animate the *card transform and custom properties*, not box-shadow.

The two materials must differ on **five axes**, not just hue, so they read as different physical substances:

| Axis | Warm Glass Copper (`#c98a5e` copper, `#8b7bff` violet, `#e8b27d` amber) | Cold Prism Frost (`#5fd4e0` cyan, `#cfe2ee` ice, `#8b9bff` periwinkle) |
|---|---|---|
| **Foil gradient** | Slow, broad linear/`conic-gradient` copper→amber→violet; warm metallic, low frequency | Tight, high-frequency prismatic `conic-gradient` cyan→ice→periwinkle; sharp spectral split |
| **Blend mode** | `overlay` + `soft-light` (warmer, "lit glass," softer contrast) | `color-dodge` (brighter, sharper, more "icy prism" glints) |
| **Glare** | Wide, soft radial; warm-white core; gentle falloff (polished metal/glass) | Small, hard radial; cool-white core; high contrast (frost crystal) |
| **Sparkle/texture** | Sparse, larger flecks (hammered copper grain) | Dense, fine glitter (frost crystals), `lighten`/`color-dodge` |
| **Edge/frame** | Warm bevel: top edge `rgba(232,178,125,0.25)`, dual border with a thin amber inner line | Cool bevel: top edge `rgba(207,226,238,0.30)`, dual border with a thin cyan inner line |

```css
/* shared card scaffold */
.card { position:relative; transform-style:preserve-3d; isolation:isolate;
  transform: rotateX(calc(var(--py,0)*1deg)) rotateY(calc(var(--px,0)*1deg)); }
.card__foil, .card__glare, .card__sparkle {
  position:absolute; inset:0; border-radius:inherit; pointer-events:none; }

/* WARM GLASS COPPER */
.card--copper .card__foil {
  background:
    repeating-linear-gradient(115deg,
      #c98a5e, #e8b27d, #8b7bff, #c98a5e 0 22%);
  background-position: calc(var(--bg-x,50%)) calc(var(--bg-y,50%));
  background-size: 300% 300%;
  mix-blend-mode: overlay; opacity:.55; }
.card--copper .card__glare {
  background: radial-gradient(circle at var(--px,50%) var(--py,50%),
    rgba(255,238,220,0.55), transparent 45%);
  mix-blend-mode: soft-light; }
.card--copper {
  box-shadow:
    inset 0 1px 0 0 rgba(232,178,125,0.25),
    0 2px 4px rgba(0,0,0,0.5), 0 20px 40px -8px rgba(0,0,0,0.6),
    0 0 60px -10px rgba(201,138,94,0.18); }   /* warm glow pool */

/* COLD PRISM FROST */
.card--frost .card__foil {
  background:
    conic-gradient(from calc(var(--bg-x,0)*1deg) at 50% 50%,
      #5fd4e0, #cfe2ee, #8b9bff, #5fd4e0);
  background-size: 200% 200%;
  mix-blend-mode: color-dodge; opacity:.45; }
.card--frost .card__glare {
  background: radial-gradient(circle at var(--px,50%) var(--py,50%),
    rgba(235,248,255,0.6), transparent 32%);
  mix-blend-mode: color-dodge; }
.card--frost {
  box-shadow:
    inset 0 1px 0 0 rgba(207,226,238,0.30),
    0 2px 4px rgba(0,0,0,0.5), 0 20px 40px -8px rgba(0,0,0,0.6),
    0 0 60px -10px rgba(95,212,224,0.16); }   /* cool glow pool */
```

Crucially the warm card's glow pool is copper, the cold card's is cyan, and the **room behind each subtly inherits the same hue at very low opacity** (a `radial-gradient` behind the card), so the spotlight on stage matches the artifact's material.

### D. The Accent-Economy Rulebook

**Budget per viewport: 3 accent instances maximum** — 1 primary action (the next step), 1 hero number (the single most important stat), 1 keyword in prose (max once per section). Everything else is greyscale (graphite/charcoal/silver). This is the operational form of "contrast through scarcity."

**Default state for accent:** OFF. Bars, badges, dividers, panel borders, icon backgrounds, section headings — all greyscale by default (`--plate-border` white at 6–8%; text in silver `#c9ccd2`-class). Accent is *added back* only to the three sanctioned slots.

**Two-palette governance (copper-warm vs frost-cool):**
- **One palette speaks per context.** A copper-source card and its hero region use copper accents; the non-active palette is neutralized to grey within that region. Never show saturated copper and saturated cyan in the same viewport at full strength.
- **Neutralize, don't delete.** When frost isn't the active palette, render its would-be accents as desaturated silver (`color-mix(in oklab, #5fd4e0, #8a8f98 80%)`), not as a second bright color.
- **Violet/periwinkle is the shared bridge.** `#8b7bff` (copper set) and `#8b9bff` (frost set) are close enough to act as a single neutral-cool connective tone for cross-section UI (e.g., global focus ring, "selected" state) so transitions between a copper section and a frost section don't read as a palette clash.
- **Per-source theming via tokens:** set `--accent: #c98a5e` (or `#5fd4e0`) on the section/card root; every sanctioned accent slot reads `var(--accent)`. Switching source = swapping one variable, exactly how Linear/shadcn theme.

**Mapping to Blue Room's regions:**
- Three-column hero: the **center card** is the only artifact and the only place its source accent glows; left analysis panel and right reading panel are plates (greyscale, hairline borders), each allowed *one* accent number if it's genuinely the key stat.
- 7-section dossier: each plate is greyscale; per section, at most one accent keyword + one accent figure, themed to whichever source that section documents.

### E. Glow Semantics & the "Rare Artifact" State

**Premium glow recipe** (large radius, low opacity, desaturated, slow):
```css
.artifact--glow {
  /* layered: tight core + wide atmospheric pool, color-mixed low */
  box-shadow:
    inset 0 1px 0 0 var(--artifact-edge-light),
    0 20px 40px -8px rgba(0,0,0,0.6),
    0 0 40px -6px color-mix(in oklab, var(--accent), transparent 84%),
    0 0 120px -20px color-mix(in oklab, var(--accent), transparent 88%);
}
```
Opacity of the color layers stays in the ~8–20% range (color-mix 80–92% transparent). Radius scales with the object: the wide pool's blur ≈ 1.5–3× the card's short dimension.

**Breathing animation** — animate opacity of a glow pseudo-element on a slow ease cycle (4–7s), never animate `box-shadow` directly:
```css
.artifact--rare::after {
  content:""; position:absolute; inset:-40px; z-index:-1; border-radius:inherit;
  background: radial-gradient(closest-side,
    color-mix(in oklab, var(--accent), transparent 82%), transparent 70%);
  animation: breathe 6s ease-in-out infinite;
}
@keyframes breathe { 0%,100%{opacity:.5} 50%{opacity:.9} }
@media (prefers-reduced-motion: reduce){ .artifact--rare::after{animation:none} }
```

**"Developed into a rare artifact" via depth, not clutter:** a paid/rare card should differ from a common one by (1) a *deeper, wider* shadow stack, (2) a *brighter lit top edge*, (3) a *larger, breathing* glow pool in the source hue, and (4) a richer foil (denser sparkle, higher-contrast glare) — NOT by adding particles, multiple neon colors, or animated borders. Restraint is the luxury signal: Vercel and Linear earn "expensive" by what they omit.

## Recommendations

**Stage 1 — Establish the three planes (do this first).** Implement the §A token set and the three classes (`.room`, `.plate`, `.artifact`). Verify the instant-read test: squint at the hero — the card should clearly float, the panels should sit flat, the page should feel like a lit room. **Benchmark to advance:** a viewer can name which element is "closest" within ~1 second without color cues. If plates still look like they float, reduce their background step toward the room and remove any outer shadow (per shadcn's "3–5% bump max").

**Stage 2 — Port the shadow/border tokens.** Adopt the Radix dark `--shadow-1…6` structure (hairline ring + graduated black + negative spread) for any raised/floating UI; keep default dossier plates on border-only. Tint all shadows toward pure black, never grey. **Threshold to change approach:** if shadows look muddy, your shadow color isn't dark enough relative to `#0a0b0d` — push to `rgba(0,0,0,…)` and increase layer count, don't increase opacity of a single layer.

**Stage 3 — Build the card material system.** Implement the pointer-tracked scaffold and the two material variants (§C). Differ them on all five axes (foil frequency, blend mode, glare softness, sparkle density, edge hue). **Benchmark:** with color temporarily desaturated, the two cards should still feel like different *materials* (different glare/texture behavior). If they look identical in greyscale, the blend-mode and glare differences aren't strong enough.

**Stage 4 — Impose the accent economy.** Audit every screen: count accent instances; if >3, cut to 1 action + 1 number + 1 keyword. Strip accent from all borders/bars/badges. Implement per-source `--accent` theming and the neutralize-the-inactive-palette rule. **Threshold:** on any viewport, if your eye doesn't land on the single intended action within ~2 seconds, you have too many accents competing.

**Stage 5 — Add glow only to artifacts and rare/paid states.** Use the large-radius/low-opacity/slow-breathing recipe; gate it behind `prefers-reduced-motion`. **Change trigger:** if a glow reads as "neon," reduce saturation (raise color-mix transparency toward 88–92%) and increase radius — premium glow is wide and faint, never tight and bright.

## Caveats
- **Linear's exact surface/border/shadow values are not officially published.** `#08090a` (base) and `#5e6ad2` (accent) are independently corroborated (Mobbin color-extraction, Linear's brand page); the panel hex `#0f1011`, the transparent-white border tiers, and all Linear shadow values come from community replica design files and should be treated as close approximations, not verbatim shipped CSS. The Radix and shadcn values, by contrast, are from those projects' own published source/docs.
- **Spotify green has two values in circulation:** #1DB954 (core brand green) and #1ED760 (lighter logo/UI tint). Either reads as "the action color" against #121212; pick one and use it only for the single sanctioned action, mirroring Spotify's own restraint.
- **Material's overlay percentages assume a `#121212` base**; Blue Room's deeper `#0a0b0d` will need slightly larger lightness steps to achieve the same perceived separation, but keep them small to preserve the luxury mood.
- **Blend modes are background-dependent.** `color-dodge`/`overlay` foils were tuned for the artwork beneath them; on very dark card art they can blow out or wash out. Test each material against real card images and adjust layer opacity (the `opacity:.45–.55` values are starting points).
- **Performance:** layered box-shadows and blend-mode stacks are paint-heavy. Don't animate `box-shadow`; animate opacity/transform on pseudo-elements and promote with `will-change: transform`. Test the holo/glare on mid-range mobile.
- **Forward-looking note:** several cited CSS technique articles are dated 2026 and describe "current" trends (inverted skeuomorphism, liquid-glass rim lighting); these are stylistic commentary, not standards — treat the specific rgba/edge-lighting recipes as patterns to validate visually, not as authoritative specs.
- **Accessibility:** desaturated accents on near-black can fall below WCAG 4.5:1 for text. Reserve true accent hues for large text/graphics/non-text indicators, and verify any accent-on-dark *text* (e.g., a hero number) meets contrast — lighten the accent for that use if needed (as shadcn does, lightening primary for dark mode).
