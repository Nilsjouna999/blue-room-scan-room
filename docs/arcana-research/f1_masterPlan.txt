## The three governing decisions

**1. INDEX vs DETAIL SPLIT — the reading page is never abridged.**
The reading page keeps every draw's full meaning + reversed + keywords + tag inline, exactly as now. No teaser, no expand/collapse, no gating — that reads as a paywall and breaks canon. The detail page adds *only what doesn't fit a single-sitting 8-draw read*: the `systemNote`, a dek line (first sentence of `meaning`, derived), adjacent-in-sequence, correspondences, a provenance stamp, and the mark at hero size. Nothing on the reading page today is ever removed or hidden behind an interaction. (Locks sub-systems 1, 2, 6.)

**2. THE RIGHT-SIDE ELEMENT — marginal deep-link + inline mark, NOT a rail (resolved conflict).**
Sub-systems 1 and 5 spec an 88–110px sticky 2-column grid rail; sub-systems 7 and 10 reject it as a second visual system forced into a single-column ~60ch reading. **Lead call: ship the marginal pattern, defer the rail.** Each draw gets (a) a right-aligned mono `EXPLORE <NAME> →` link outside the prose measure (a real `<a href>`, reflowing inline below the meaning under 900px), and (b) a small identity mark beside the slot label, with a 6px ember dot at its corner when `reversed` is non-empty. This answers the client's #2 and #3 today, reuses existing IBM Plex Mono / ghost-grey tokens (zero new visual language), carries zero layout risk, and is fully reversible. The fixed-width sticky rail is **out of scope for v1** — revisit only if a dedicated measure/layout pass proves the reading can spare the width.

**3. THE IDENTITY SYSTEM — "Sealed Marks," procedural now, GPT-art-swappable later.**
Every one of the 154 results gets a two-register mark: a **system emblem** (top — the real traditional glyph: zodiac ♈–♓, tarot numerals 0–XXI, Elder Futhark runes, trigrams ☰–☷, hexagram number 1–64, year-animal kanji, life-path numeral) fused with an **entry cipher** (bottom — a deterministic 3×3 dot matrix hashed from the entry name). Ships as code day one (`arcana-marks.js`, `currentColor`, ghost at rest → gold on focus), zero art dependency, zero collisions, regeneratable from the name. The mark lives in a **fixed-size slot** so the eventual GPT-authored bespoke SVG (system token + name-seeded cipher, one brief → 154 outputs) drops in with no layout retrofit. (Locks sub-systems 3, 5, 10.)

**Locked URL contract (one source of truth, or links die):** reuse the existing mount switch —
`index.html?dev=arcana-entry&system=<key>&slug=<slug>&from=<seed>`
where `system` ∈ the 7 `indexCodex()` keys (`sun, chinese, lifePath, tarotMajor, rune, trigram, hexagram`), `slug = slugify(name)`, `from` = seed (optional; its absence just omits the back-to-reading link), and the two tarot draws disambiguate with `&role=counsel`. The reading page *constructs* this and the detail page *parses* it — both import the same map and the same `slugify()`.
