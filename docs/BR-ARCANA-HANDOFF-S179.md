## 0. What this session changed (ground truth)

The Arcana product is a static HTML/CSS/JS app (no build step for the app itself; `node` not
installed). Routing is `?dev=<name>` handled by `mountDev` in `app.js` (allowlist ~line 94);
the live menu (`renderMenu` in `app.js`) links out to surfaces; `codex.html` is a standalone
page. Modules load via `<script>` in `index.html` and expose `window.BR*` globals.

**The Reliquary — profile/hub (`?dev=profile`, `arcana-profile.js` + `.css`)** — the account home:
- Top-to-bottom: **identity name + crown (ONE left-aligned block above the Rings)** → Rings →
  Vault → Showcase → Friends → Referral.
- **Identity name** = "Antton Aikio" in **white with an animated black "oil/ink" edge** that
  ripples around the letters (SVG filter `#pfOil`, rim-only displacement; reduced-motion → plain
  white). The old "Twice-Kindled Founder" nitrogen headline was scrapped earlier and is gone.
- **Crown** = the reading button (placeholder forged-SVG; final art from GPT later). Its spine
  (Aries · Snake · Life Path 7) + "3 rubies set…" + **"Open this reading"** sit left of it.
  **No profile-side redraw** — a new reading is drawn from the Arcana Room.
- **Rings** = only the "Read for someone" box (opens a 12-relation picker → input room). No
  example rings.
- **Vault** = minted-card SLOTS only, **empty ("not open yet")** — real profile vault not built.
  **Nothing on the profile links to the main/scan `?dev=vault`** (masthead Vault door removed too).
- **Showcase** = clickable SLOTS. Click → picker of the seeker's OWN reading results (Reading /
  Sun sign / Year animal / Life path / Tarot) + minted cards ("none yet"); choosing fills the slot.
- Menu has a **"◆ The Reliquary"** pill → `?dev=profile` (BR-S173).
- Paid controls are **violet**; free "Open this reading" is gold. Gold muted globally (`#a2864a`).

**The Arcana Reading — the FULL reading (`?dev=arcana-reading`, `arcana-reading.js` + `.css`)**:
- Birth-derived draw across 8 systems **+ per-draw record pages** ("Open the record": Source ·
  Essence · Gifts · Strengths & Shadow · Hardships · In love · Compatibility · Origin ·
  Attributions · Correspondences · Deeper · Contested + the Soul-Signature Bloom / Wu Xing /
  decan / numerology diagrams). Record pages are hash-routed (`#/e/key/slug/seed`).
- **Reads `?seed=birth~name~y~m~d`** (default with no seed = Antton Aikio, 9 Apr 2001, Inari →
  Aries · Snake · Life Path 7 · Emperor → "The Twice-Kindled Founder").
- Header has **"← Profile"** (→ `?dev=profile`) + "Blue Room · Arcana" + "Draw again".
- **GENERATED** by `python arcana-build/build_inapp.py` from `arcana-build/gen_body.js` +
  `gen_head.css` + `fonts.css`, data inline→fetch of `codex-data.json` +
  `arcana-build/{kb_compact,practical,kwcolor}.json`. **Do NOT hand-edit `arcana-reading.js/.css`.**
- Replaced the scrapped "Armor Received" (`arcana-result`, deleted BR-S172) — don't reintroduce.

**The intake — "The Setting of Marks" (`?dev=arcane`, `arcane.js` + `arcane.css`)**:
- Marks: name · day (birth date) · hour · ground · matter. On draw it **parses the birth date**
  (D-M-Y free text / ISO / month names) and hands the reading `?seed=birth~name~y~m~d` → the
  reading derives correctly (unparseable date → name-seeded draw).
- **For whom**: Myself · Family · A friend. **First reading is for yourself**; Family + Friend are
  **locked until a reading is completed** (persisted via `localStorage("br_has_reading")`, set when
  a reading is drawn; also unlocked when arriving from the profile's "read for someone"). Family
  (unlocked) opens an 11-member picker; a single dull note explains the lock (no emojis/logos).
- Intake → ceremony (`ceremony.js`) → the reading. Ceremony/intake exits go to `?dev=arcana-reading`.

**The Codex (`codex.html`, generated by `python build_codex.py` from `codex-data.json`)**:
- 166 entries across 8 systems + the codex search (left). **NEW: a second search bar far right —
  "define an English word…"** — an English dictionary using the free **api.dictionaryapi.dev**
  (Enter → phonetics + up to 3 POS × 2 definitions in a popover). **Needs network**; the codex
  search itself stays offline/static. Regenerate via `build_codex.py`.
